## Section 1: Polish and UX

### 1.1 Colored terminal output

**Purpose:** Colored tables for `hub info` and `motor` queries when
stdout is a TTY. Gate on `std::io::IsTerminal`.

**Priority:** LOW

### 1.2 `--wait` flag for motion commands

**Purpose:** After a fire-and-forget motion command (`open`, `close`,
`set-position`, etc.), optionally poll `motor_position` until the motor
reaches the target. Useful for scripts that need "wait until done".

**Priority:** MEDIUM

### 1.3 `--timeout` override

**Purpose:** Library hardcodes `DEFAULT_QUERY_TIMEOUT = 2500ms` and
`BROADCAST_COLLECT_WINDOW = 3000ms`. Some LANs need longer. Requires
the library to expose timeout configurability first (tracked in the
library's project-tracker).

**Priority:** LOW — blocked by library change

### 1.4 `--verbose` / `-v` flag

**Purpose:** Print trace-level info to stderr: "connecting to
192.168.5.234:1487", "name resolution: 'kitchen' matched 3YC",
"writing bytes `!3YCc;`". Don't need a logging framework —
`eprintln!`-gated on a bool flag is sufficient.

**Priority:** MEDIUM

### 1.5 Per-motor batched query in the library

**Purpose:** `motor <motor>` (no sub-query) currently calls
`hub.info()` and filters. A dedicated `hub.motor_info(addr)` batched
query in the library would avoid the broadcast-window latency. Tracked
here because it requires a library change.

**Priority:** LOW
