## Section 1: WebSocket RPC channel

### 1.1 WebSocket RPC support

**Purpose:** Rooms, scenes, timers, real-time position push updates,
battery percentage. Not available over the ASCII channel.

**Current Progress:** Not started. ASCII-only in v1.

**Priority:** MEDIUM — ASCII covers the common case; WebSocket only
matters for state-mirroring integrations (e.g. Home Assistant).

#### Tasks

1. **Add `websocket` Cargo feature**
   - New deps gated: `tokio-tungstenite`, `rustls`, `webpki-roots`
   - Hub cert is `CN=AWS IoT Certificate` → cert validation must be
     off (cert is not minted for hub LAN IP)

2. **New transport module parallel to `transport.rs`**
   - Line-delimited JSON over WS
   - Port `aiopulse2/devices.py::Hub.run()` and `_process_response()`

3. **New public types**
   - `Room`, `Scene`, `Timer` structs
   - `MotorSnapshot` variant with richer fields

4. **New public API**
   - `subscribe_position_updates() -> impl Stream<Item = ...>`
   - `list_rooms()`, `list_scenes()`, `list_timers()`
   - `trigger_scene(&scene_id)`, `activate_timer(&timer_id)`

5. **Docs**
   - Example showing push-update subscription
   - README section on feature-flag trade-offs

#### Code References

- `aiopulse2/devices.py` — reference implementation
- `PULSE_PRO_LOCAL_API.md` §4 — what we know about the WS channel
